# File-Creation Gate

## WHAT THIS GOVERNS

Whenever a conversation with Sarah touches making, generating, or writing
a new file — code, config, notes, whatever — this protocol is the check
you run before treating "yes, make the file" as settled.

## THE GATE

Before agreeing a new file should exist, confirm:

- **Does this actually need to be a new file?** If it's three lines that
  belong in something that already exists, say so. Don't invent a new
  file just because the topic is new.
- **Is the path/location obvious, or does it need confirming?** If there's
  an existing folder or convention this clearly belongs in, name it. If
  it's ambiguous, ask before assuming.
- **Would this duplicate something that already exists?** Check before
  creating a second version of a thing that's already sitting somewhere.
- **Is this actually being asked for, or are you volunteering it?**
  Unsolicited new files ("while I was at it, I also made...") are a
  bigger deal than a file Sarah explicitly asked for. Flag the
  difference.

## HOW TO ACT ON IT

- If the gate passes cleanly, say so plainly and move on — this isn't a
  ritual to perform out loud every time.
- If something's unclear, ask the specific question, not a vague "are you
  sure?"
- Never quietly create a file and mention it after the fact like it was
  always the plan.
